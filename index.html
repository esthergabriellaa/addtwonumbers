Add Two Numbers

Two numbers are hiding inside linked lists — written backwards, one digit per node. Your job: add them up without ever converting to integers.

Medium Linked List Math

What are we actually dealing with?

You're given two non-empty linked lists representing two non-negative integers. The digits are stored in reverse order — the 1s digit comes first.

For example, the number 342 is stored as the list 2 → 4 → 3. The number 465 is stored as 5 → 6 → 4. Your task is to add them and return the result as a linked list — also in reverse.

Notice the twist: because the lists are reversed, the 1s digit is at the front. That's actually a gift — it means we can walk both lists from left to right and add column by column, just like elementary school addition.

Think back to grade school

Remember how you add numbers on paper? You line them up, start from the right, and add each column. If the sum is 10 or more, you carry the extra 1 to the next column.

Step through the addition column by column.

We'll add 342 + 465 one column at a time. Columns go right-to-left in normal addition, but notice — our linked lists give us the digits right-to-left already, so we go left-to-right!

That last point is the key insight. In a linked list, we always start at the head. Since the digits are stored in reverse, the head gives us the 1s digit — exactly where normal addition starts. Reversed order is actually the helpful order.

Walking through the algorithm

Here's the full procedure in plain English before we look at code:

Use two pointers — one per list — and walk them forward in lockstep. At each step, add p1.val + p2.val + carry. The 1s digit of the result becomes a new node; the 10s digit (0 or 1) becomes the carry for the next step.

The carry can only ever be 0 or 1, because the max possible sum at any step is 9 + 9 + 1 = 19 — and 19 / 10 = 1 remainder 9.

One edge case: after both lists are exhausted, the carry might still be 1. If so, create one final node with value 1. Example: 999 + 1 = 1000 — four digits from two three-digit numbers.

Edge cases to watch for

Lists of different lengths

What if one list runs out before the other? Just treat the missing digit as 0. The loop continues until both pointers are null and carry is 0.

Leftover carry at the end

Adding 999 + 1 gives 1000 — but after processing all nodes, your carry is 1 with no nodes left. Don't forget to create that final node.

The solution

The code is a direct translation of everything above. A dummy head node keeps things clean — we never have to special-case the first node.

PYTHON 
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

def addTwoNumbers(l1: ListNode, l2: ListNode) -> ListNode:
    dummy = ListNode()   # placeholder head
    curr  = dummy
    carry = 0

    while l1 or l2 or carry:
        v1 = l1.val if l1 else 0   # treat exhausted list as 0
        v2 = l2.val if l2 else 0

        total = v1 + v2 + carry
        carry = total // 10          # 0 or 1
        digit = total %  10          # the digit we keep

        curr.next = ListNode(digit)
        curr = curr.next

        if l1: l1 = l1.next
        if l2: l2 = l2.next

    return dummy.next

Why a dummy head?

Without the dummy node, you'd need an if result is None: result = ListNode(digit) check on the first iteration. The dummy node lets every iteration look identical — just curr.next = ListNode(digit). At the end we return dummy.next to skip the placeholder.

Complexity analysis

time complexity
O(n)
We visit each node at most once. n = length of the longer list.
space complexity
O(n)
The result list is at most n+1 nodes long (the extra 1 for a final carry).
